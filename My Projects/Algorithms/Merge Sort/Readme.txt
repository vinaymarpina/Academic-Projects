
-----------------------------------------------------------------
Instructions:
-----------------------------------------------------------------
Project code requires minimum java version 1.5

Compile : javac MergeSort.java

Run: java MergeSort

     By limiting heap memory--

     java -Xms64m -Xmx256m MergeSort
     java -Xms64m -Xmx512m MergeSort

-----------------------------------------------------------------
Expected Result: 
-----------------------------------------------------------------

Statistics of running three implementaions of mergesort
as given in the requirement on standard output.

1) Number of integers,Execution time and memory consumption(Difference of Runtime.freeMemory()) for
   each iteration will be printed out.

2) Negative memory differences indicate garbage collection activity of JVM.

3) Random numbers are generated using Collections.shuffle();

-----------------------------------------------------------------
Analysis: 
-----------------------------------------------------------------

1)All implementations showing O(nlogn) running time.

2)Dynamic memory allocation implemention of mergesort in java seems to run
  faster than second implementation for large arrays(large numbers)
  while second one is faster for small arrays(in range 32768-524288).

3)Third implementation beats other two by fair margin 
  (10-40% speedups for small numbers, ~20% for large numbers).
